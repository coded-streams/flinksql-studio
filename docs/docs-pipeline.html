◈ What is the Pipeline Manager?

The Pipeline Manager is a visual canvas where you build Flink SQL pipelines without writing SQL by hand. Drop sources, transformations, and sinks, connect with typed edges, configure each node via a modal dialog, then click Submit to generate and run the complete Flink SQL.

🖱

Drag & Double-click

Drag operators from the palette. Double-click any node to open its configuration modal. Modal opens next to the node, is draggable, and can be reopened any number of times.

🔐

Auth on All Connectors

Kafka SASL/SSL, Schema Registry credentials, JDBC username/password, Elasticsearch basic auth — optional fields on every connector operator.

🤖

AI Model Operator

Connect your pipeline to any hosted AI model — OpenAI, Anthropic, Bedrock, Vertex AI, Hugging Face — via a UDF or Flink 2.1+ ML_PREDICT.

⨍

My UDFs

UDFs registered in UDF Manager appear in the My UDFs palette group and can be dropped as pipeline operators with a dropdown selector.

⚡

Live SQL Preview

Right panel shows full generated SQL updating in real time, including SET statements, auth config, DDL, and INSERT INTO.

▶

Animation & Errors

Run simulates data flow with animated particles. Validation errors highlight nodes in red with a dismissable banner listing each problem.

⊙ Opening & Tabs

Click ◈ Pipeline in the topbar. Three tabs appear:

  • ◈ Builder — canvas, operator palette, live SQL preview.
  • ⟨/⟩ SQL — full generated SQL with Copy and Insert buttons.
  • 📁 Saved — saved pipelines with load, export, delete actions.
💡
Fullscreen Click ⤢ top-right to expand to full screen. Click again to return.

☰ Operator Palette

The left sidebar lists all operators in collapsible groups. Badges:

  • S (amber) — Stateful: requires checkpointing configuration.
  • ⚡ (yellow) — Requires an external connector JAR on the Flink classpath.

Below operators, the Edge Types section lets you pick the partitioning strategy before drawing a connection.

⚙ Configuring Nodes

Double-click any node to open its modal. Two tabs:

  • ⚙ Parameters — connector banner (if Systems Manager has a matching connector), checkpointing (stateful ops), node label, description, colour, and all operator-specific fields. Auth fields are always optional.
  • ℹ About — what the operator does, when to use it, tips.
1

Double-click the node

Config modal opens positioned next to the node. Drag its header to move it.

2

Fill required fields

Required fields have a red asterisk. Auth fields (SASL, passwords, API keys) are optional — leave blank for unauthenticated connections.

3

Click ✓ Apply

Node saves, modal closes, status changes to "✓ ready". Live SQL panel updates immediately.

4

Reopen any time

Double-click again to edit. All previously saved values are pre-filled.

💡
Schema format: Enter one column per line with no commas: id BIGINT, amount DOUBLE, ts TIMESTAMP(3). The SQL generator adds commas automatically.

Kafka Authentication & Schema Registry

All Kafka operators include optional auth fields. Leave blank for unauthenticated local Kafka.

Auth fields on all Kafka operators: Security Protocol — PLAINTEXT / SSL / SASL_PLAINTEXT / SASL_SSL
SASL Mechanism — PLAIN / SCRAM-SHA-256 / SCRAM-SHA-512 / GSSAPI
SASL Username / API Key — Confluent Cloud API key or LDAP username
SASL Password / Secret — Confluent Cloud API secret or LDAP password
SSL Truststore Path — Container path to JKS file e.g. /etc/kafka/truststore.jks
Schema Registry fields: Schema Registry URL — e.g. http://schema-registry:8081 or Confluent Cloud SR URL

🤖 AI Model Operator

The AI Model operator connects your Flink pipeline to any hosted AI or ML inference endpoint. It generates a CREATE TEMPORARY VIEW containing a UDF call (or ML_PREDICT on Flink 2.1+) that applies inference to each row in the stream.

⚠
Flink SQL cannot call HTTP APIs directly. You must register an inference UDF first — via Otter Streams or a custom Java UDF. On Flink 2.1+, the native ML_PREDICT function is also available. Set API keys as Flink cluster environment variables — never hardcode secrets in SQL.

-- AI Model: gpt-4o-mini via OpenAI
-- Register UDF first:
-- CREATE TEMPORARY FUNCTION fraud_score AS 'com.your.AIUdf' LANGUAGE JAVA;
CREATE TEMPORARY VIEW ai_scored AS
SELECT
  id, ts,
  fraud_score(event_payload) AS ai_result
FROM source_table;
-- Flink 2.1+ native inference (no UDF required for supported models)
CREATE MODEL my_model
INPUT (payload STRING)
OUTPUT (prediction STRING)
WITH (
  'task'           = 'classification',
  'openai.endpoint'= 'https://api.openai.com/v1',
  'openai.api_key' = '${env:OPENAI_API_KEY}'
);

SELECT id, ts,
  ML_PREDICT('my_model', payload) AS prediction
FROM events;
  • OpenAI / Anthropic / Cohere — Bearer token. Set API key as env var in Flink config. Use Otter Streams HTTP module or Flink 2.1+ ML_PREDICT.
  • AWS Bedrock — AWS SigV4 signing. Mount IAM credentials on TaskManager containers. Model IDs: amazon.nova-pro-v1:0, anthropic.claude-3-5-sonnet-20241022-v2:0.
  • Google Vertex AI — OAuth2 service account. Mount GCP credentials JSON. Model IDs: gemini-1.5-flash-001, text-bison@002.
  • Azure OpenAI — Azure AD token or API key. Per-deployment endpoint: https://<resource>.openai.azure.com/deployments/<name>/chat/completions.
  • Hugging Face — Bearer token from HF settings. Use Inference Endpoints URL for production (not the free shared API).
  • Otter Streams UDF — Upload shaded JAR via UDF Manager, register with CREATE TEMPORARY FUNCTION, select in the AI Model operator. Supports ONNX, TensorFlow, PyTorch, XGBoost, and HTTP remote inference.

⨍ My UDFs Group

Any UDF registered via ⨍ UDF Manager appears in the My UDFs palette group. Drag UDF Function onto the canvas, double-click it, and select your function from the dropdown. Configure input columns, output alias, and passthrough columns.

💡
No UDFs showing? Go to ⨍ UDF Manager → Upload JAR → complete Steps 1–3 (ADD JAR + CREATE FUNCTION). Return to Pipeline Manager — the UDF appears in the dropdown automatically.

▶ Animation & Error Handling

Click Run in the toolbar to simulate data flow with animated particles. Validation runs first — all required params must be filled and all transform nodes must be connected.

When validation fails: a red error banner appears at the top of the canvas listing each error with node name and a Locate button. The floating ⏹ STOP ANIMATION button at the top-center always stops particles immediately. Clicking ✕ Clear & Close on the banner clears all error highlights.

💡
Run becomes Stop while animating. The toolbar Run button changes to ⏹ Stop (red). The floating stop button is always available as a fallback.

⟨/⟩ SQL Generation

The generator produces complete, runnable Flink SQL including SET statements, connector warnings, source DDL with auth config and Schema Registry settings, sink DDL, and INSERT INTO from the transform chain.

⚠
Auth in generated SQL. Passwords and API keys are written directly into the generated SQL. For production, consider Flink secrets management or environment variable substitution rather than hardcoding credentials.

② Tutorial 2 — Confluent Cloud Kafka Pipeline

Kafka source (Confluent Cloud, SASL_SSL + Schema Registry) → Filter → Kafka sink.

Kafka Source — double-click and fill in:

  • Table Name: cc_events, Topic: raw-events, Format: avro-confluent
  • Bootstrap: pkc-xxxxx.us-east-1.aws.confluent.cloud:9092
  • Security Protocol: SASL_SSL, Mechanism: PLAIN
  • SASL Username: <API_KEY>, Password: <API_SECRET>
  • SR URL: https://psrc-xxxxx.confluent.cloud, SR User: <SR_KEY>, SR Pass: <SR_SECRET>
  • Schema: user_id STRING / event STRING / ts TIMESTAMP(3), Watermark: ts

Filter: event != 'heartbeat'

Kafka Sink: cc_filtered / topic filtered-events — same SASL_SSL fields.

CREATE TABLE IF NOT EXISTS cc_events (
  user_id STRING,
  event   STRING,
  ts      TIMESTAMP(3),
  WATERMARK FOR ts AS ts - INTERVAL '5' SECOND
) WITH (
  'connector'                           = 'kafka',
  'topic'                               = 'raw-events',
  'properties.bootstrap.servers'        = 'pkc-xxxxx.confluent.cloud:9092',
  'properties.group.id'                 = 'flink-group',
  'scan.startup.mode'                   = 'latest-offset',
  'format'                              = 'avro-confluent',
  'properties.security.protocol'        = 'SASL_SSL',
  'properties.sasl.mechanism'           = 'PLAIN',
  'properties.sasl.jaas.config'         = 'org.apache.kafka.common.security.plain.PlainLoginModule required username="API_KEY" password="API_SECRET";',
  'schema-registry.url'                 = 'https://psrc-xxxxx.confluent.cloud',
  'schema-registry.basic.auth.user.info'= 'SR_KEY:SR_SECRET'
);

③ Tutorial 3 — Real-time AI Fraud Scoring

Kafka source → Filter → AI Model (Otter Streams XGBoost) → Kafka sink.

1

Register the inference UDF

⨍ UDF Manager → Upload JAR (Otter Streams shaded JAR) → Register: name fraud_score, class com.codedstreams.otter.MLInferenceUDF, LANGUAGE JAVA. Confirm with SHOW USER FUNCTIONS.

2

Build the pipeline

Kafka Source → Filter (amount > 50.0) → AI Model → Kafka Sink.

3

Configure AI Model

Double-click AI Model: Provider = Otter Streams UDF, Model = fraud-xgboost-v2, Input Column = amount, Output Alias = fraud_probability, Passthrough = tx_id, ts, UDF Name = fraud_score. Apply.

4

Submit

Click Submit. The generated SQL calls fraud_score(amount) AS fraud_probability inside a CREATE TEMPORARY VIEW and inserts results into the Kafka sink.
