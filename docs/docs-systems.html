⊙ What is Systems Manager?

The Systems Manager (⊙ Systems in the topbar) is the central registry for all external connections your Flink pipelines will use. You register connection details once — bootstrap servers, JDBC URLs, credentials, endpoints — and the Pipeline Manager can then auto-fill those values into any compatible operator node.

Connection Registry

Store named connector configurations (Kafka, JDBC, ES, S3, Pulsar, MongoDB, Kinesis) in browser localStorage.

Pipeline Prefill

In the Pipeline Manager, compatible operator nodes show a connector banner. Click "Use" to auto-fill all connection parameters from the registry.

JAR Tracking

See which connector JARs are required for your registered connectors and get direct download links for the correct versions.

Environment Awareness

Tag connectors as Dev, Staging, or Production. Color-coded badges show environment at a glance.

⊙ Opening & Interface

Click ⊙ Systems in the topbar. The Systems Manager modal opens with two main areas:

  • Left panel — list of all registered connectors with type icon, name, environment badge, and quick-test button.
  • Right panel — connector detail form for adding or editing a connector.

Connector data is saved to localStorage('strlabstudio_systems') and persists across page reloads.

＋ Adding a Connector

1

Click + Add Connector

In the left panel, click the Add button. The right panel shows an empty connector form.

2

Select connector type

Choose from the type dropdown: Kafka, JDBC, Elasticsearch, S3/MinIO, Pulsar, MongoDB, Kinesis, or Custom. The form adjusts to show the relevant fields for that connector type.

3

Fill in connection details

Enter the name, environment (Dev/Staging/Production), and all connector-specific fields (bootstrap servers, JDBC URL, etc). See Field Reference below.

4

Save

Click Save. The connector appears in the left panel list. It is now available for Pipeline Manager auto-fill.

⚡ Pipeline Manager Prefill

When you drop a connector-aware operator onto the Pipeline Manager canvas and double-click to configure it, the inline form shows a connector banner at the top if a matching connector is registered in Systems Manager. Click Use next to a connector name to auto-fill all compatible fields instantly.

💡
Connector matching The Pipeline Manager matches connectors by type keyword in the name — e.g. a connector named "prod-kafka-cluster" matches Kafka source and sink nodes. Name your connectors descriptively for better auto-matching.

If no matching connector is registered and the operator requires a JAR, a warning banner with a link to the Systems Manager appears instead.

📦 JAR Management

Each connector type that requires an external JAR displays the correct Maven artifact name and a download link. The required JARs vary by Flink version — always match the connector JAR version to your Flink cluster version.

🚨
Version mismatch causes NoClassDefFoundError A Kafka connector built for Flink 1.18 will not work on Flink 1.19 or 2.0. Always download the connector JAR that matches your exact Flink version.

After downloading, place JARs in /opt/flink/lib/ on all JobManager and TaskManager containers (or use the UDF Manager's Upload JAR feature for session-scoped loading).

⬡ Apache Kafka

Prerequisites

  • Kafka broker reachable from Flink TaskManagers (not just from your laptop).
  • Flink Kafka connector JAR on the classpath: flink-sql-connector-kafka-3.3.0-1.19.jar (adjust for your Flink version).

Confluent Cloud Kafka

-- Confluent Cloud connection with SASL_SSL
'connector'                       = 'kafka',
'topic'                           = 'my-topic',
'properties.bootstrap.servers'    = 'pkc-xxxxx.us-east-1.aws.confluent.cloud:9092',
'properties.security.protocol'    = 'SASL_SSL',
'properties.sasl.mechanism'       = 'PLAIN',
'properties.sasl.jaas.config'     = 'org.apache.kafka.common.security.plain.PlainLoginModule required username="API_KEY" password="API_SECRET";',
'format'                          = 'json'

🗄 JDBC / Databases

Prerequisites

  • Flink JDBC connector JAR: flink-connector-jdbc-3.1.2-1.17.jar
  • Database driver JAR (e.g. postgresql-42.7.1.jar for PostgreSQL)
  • Both JARs must be in /opt/flink/lib/ on all cluster containers
'connector'  = 'jdbc',
'url'        = 'jdbc:postgresql://postgres:5432/mydb',
'table-name' = 'public.orders',
'username'   = 'flink_user',
'password'   = 'secret'
'connector'  = 'jdbc',
'url'        = 'jdbc:mysql://mysql:3306/mydb',
'table-name' = 'orders',
'username'   = 'flink_user',
'password'   = 'secret',
'driver'     = 'com.mysql.cj.jdbc.Driver'
'connector'  = 'jdbc',
'url'        = 'jdbc:sqlserver://sqlserver:1433;database=mydb',
'table-name' = 'dbo.orders',
'username'   = 'sa',
'password'   = 'Password123'

🔍 Elasticsearch

'connector' = 'elasticsearch-7',    -- or 'elasticsearch-8' for ES 8+
'hosts'     = 'http://elasticsearch:9200',
'index'     = 'my-index'
-- Note: PRIMARY KEY NOT ENFORCED required for upsert mode
⚠
ES 8 connector Use elasticsearch-8 connector identifier and the corresponding JAR version. The connector field value and the JAR version must both match your ES cluster major version.

☁ S3 / Object Storage

Flink reads and writes S3-compatible object storage (AWS S3, MinIO, GCS) using the Hadoop S3A filesystem. No separate connector JAR needed — configure the S3A credentials in Flink's config.yaml.

-- flink config.yaml (Flink 2.0+) or flink-conf.yaml (Flink 1.x)
fs.s3a.endpoint: http://minio:9000
fs.s3a.access.key: minioadmin
fs.s3a.secret.key: minioadmin
fs.s3a.path.style.access: true
-- Then in Flink SQL:
CREATE TABLE s3_sink (
  id BIGINT, payload STRING, ts TIMESTAMP(3)
) WITH (
  'connector' = 'filesystem',
  'path'      = 's3a://my-bucket/output/',
  'format'    = 'parquet'
);

① Tutorial — Register a Kafka Connector

1

Open Systems Manager

Click ⊙ Systems in the topbar.

2

Add Connector

Click + Add Connector. Select type Kafka.

3

Fill in details

Name: local-kafka, Environment: Dev, Bootstrap Servers: kafka:9092, Topic: events, Group ID: flink-group, Format: json.

4

Save

Click Save. The connector appears in the list with a Kafka icon and Dev badge.

5

Use in Pipeline Manager

Open the Pipeline Manager. Drag a Kafka Source node. Double-click to expand. The connector banner shows local-kafka. Click Use — all Kafka fields are auto-filled instantly.

② Tutorial — Register a PostgreSQL Connector

1

Ensure JARs are in place

Copy flink-connector-jdbc-3.1.2-1.19.jar and postgresql-42.7.1.jar to /opt/flink/lib/ on all containers and restart the cluster.

2

Register in Systems Manager

Open ⊙ Systems → Add Connector → Type: JDBC. Name: postgres-dev, JDBC URL: jdbc:postgresql://postgres:5432/mydb, Username: flink_user, Password: secret.

3

Use in Pipeline Manager

Add a JDBC Sink node. Double-click to expand — the connector banner shows postgres-dev. Click Use. JDBC URL is auto-filled. Enter the table name, then Apply.

4

Test with a query

In the SQL editor: CREATE TABLE test_pg (...) WITH ('connector'='jdbc','url'='jdbc:postgresql://postgres:5432/mydb','table-name'='test'); then INSERT INTO test_pg SELECT ....
