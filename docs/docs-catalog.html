Catalog Management

⊟ Flink Catalog Model

Apache Flink organizes metadata in a three-level hierarchy:

📦 catalog_name
🗄 database_name
📋 table_name
🔹 column_name : TYPE

Every Flink SQL session has a current catalog and current database. Unqualified table references (e.g. SELECT * FROM orders) resolve against the current catalog and database. Fully qualified references (my_catalog.my_db.orders) always work regardless of current context.

LevelSQLDescription
CatalogSHOW CATALOGSTop-level namespace. Multiple catalogs can coexist.
DatabaseSHOW DATABASESGroups tables within a catalog.
Table/ViewSHOW TABLESTables and views within a database.
ColumnDESCRIBE table_nameColumn names, types, nullability, computed columns, watermarks.

⊕ Default Catalog

Every Flink session starts with the default_catalog containing a single default database. Tables created without specifying a catalog live here. They are session-scoped (TEMPORARY) unless explicitly registered with a persistent catalog.

-- Always available after connecting:
SHOW CATALOGS;      -- returns: default_catalog
SHOW DATABASES;     -- returns: default
SHOW TABLES;        -- returns: tables created in this session

-- Switch context:
USE CATALOG default_catalog;
USE `default`;

-- Create a persistent database:
CREATE DATABASE IF NOT EXISTS analytics;
USE analytics;
ℹ
Session-scoped vs persistent CREATE TEMPORARY TABLE lives only for the current session. CREATE TABLE (without TEMPORARY) is registered in the current catalog — if that catalog is persistent (Hive, Iceberg, JDBC), the table survives session restarts.

📂 Sidebar Browser

The CATALOG tab in the left sidebar shows the live catalog tree for the connected Flink session. It displays:

  • Catalogs — all registered catalogs (bold, catalog icon).
  • Databases — expandable under each catalog.
  • Tables and Views — listed under each database.
  • Columns — click any table to expand and see its column names and types.

Click any table or column name to insert it at the cursor position in the SQL editor.

⟳ Refresh & Navigation

  • ⟳ Catalog button in the topbar — refreshes the sidebar catalog tree against the live session.
  • ⟳ icon within the sidebar panel — same action.
  • The sidebar auto-refreshes after any DDL statement (CREATE TABLE, DROP TABLE, USE CATALOG, etc.).

The status bar at the bottom shows the current catalog.database context, e.g. catalog: default_catalog.default. This updates automatically when you run USE CATALOG or USE database.

✏ Insert DDL Snippets

The sidebar has two quick-action buttons above the tree:

  • ＋ Catalog — inserts a CREATE CATALOG template at the cursor.
  • ＋ Database — inserts a CREATE DATABASE IF NOT EXISTS template at the cursor.

Additionally, the Snippets library (✂ Snippets in the topbar → SETUP category) contains:

  • Use Default Catalog — USE CATALOG default_catalog; USE \`default\`;
  • Create In-Memory Catalog — full template for a generic catalog.
  • Use Hive Catalog — full Hive Metastore catalog creation template.

⊕ Catalog Manager

The ⊕ Catalogs button in the topbar opens the Catalog Manager — a guided modal for registering external catalogs without writing SQL by hand.

📦

Register Catalog

Select catalog type (Hive, Iceberg, Paimon, JDBC, Generic), fill in connection details, and click Register. The SQL is generated and executed automatically.

📋

Registered Catalogs

Lists all catalogs currently registered in the session. Click any catalog to switch to it (USE CATALOG).

🔍

Browse

After registering, the sidebar catalog tree updates automatically. Expand the new catalog to browse its databases and tables.

🗑

Drop Catalog

Remove a registered catalog from the session with confirmation. This does not delete the underlying data.

＋ Registering Catalogs

1

Open Catalog Manager

Click ⊕ Catalogs in the topbar.

2

Select catalog type

Choose from: In-Memory, Hive Metastore, Iceberg, Paimon, JDBC Catalog, or Generic.

3

Fill in connection details

Each type shows the relevant fields — Metastore URI for Hive, warehouse path for Iceberg, JDBC URL for JDBC Catalog, etc.

4

Register

Click Register Catalog. Studio generates the CREATE CATALOG SQL and executes it. The catalog appears in the sidebar immediately.

5

Switch context

Click the new catalog in the list (or run USE CATALOG catalog_name;) to switch to it. All unqualified table references now resolve from this catalog.

💾 In-Memory Catalog

The built-in Generic In-Memory catalog stores table metadata in the Flink session's JVM memory. Useful for creating persistent aliases without a real Metastore — but data is lost when the session ends.

CREATE CATALOG my_catalog WITH (
  'type' = 'generic_in_memory'
);
USE CATALOG my_catalog;
CREATE DATABASE analytics;
USE analytics;

🐝 Hive Metastore

Connect Flink to an existing Hive Metastore to read and write Hive tables, share metadata with Spark/Trino, and persist table definitions across sessions.

Prerequisites

  • Hive Metastore service running and reachable from Flink containers.
  • flink-sql-connector-hive-3.1.3_2.12-1.19.jar in /opt/flink/lib/ on all containers.
  • hive-exec-3.1.3.jar in /opt/flink/lib/.
  • hive-site.xml accessible from Flink containers (copy to /opt/flink/conf/).
CREATE CATALOG hive_catalog WITH (
  'type'              = 'hive',
  'hive-conf-dir'     = '/opt/flink/conf',      -- directory containing hive-site.xml
  'hive-version'      = '3.1.3',
  'default-database'  = 'default'
);
USE CATALOG hive_catalog;
SHOW DATABASES;   -- shows Hive databases
SHOW TABLES;      -- shows Hive tables
⚠
hive-site.xml must be on the Flink container The Flink SQL Gateway (not your laptop or the Studio browser) needs to read hive-site.xml. Mount it as a ConfigMap in Kubernetes or copy it into the container at /opt/flink/conf/hive-site.xml.

🧊 Iceberg

Apache Iceberg provides ACID-compliant, schema-evolving tables stored in object storage. Flink can read and write Iceberg tables using the Iceberg Flink runtime JAR.

Prerequisites

  • iceberg-flink-runtime-1.19-1.6.1.jar in /opt/flink/lib/
  • Object storage (S3/MinIO/HDFS) accessible from Flink containers
  • Hadoop filesystem JARs if using S3A
CREATE CATALOG iceberg_catalog WITH (
  'type'            = 'iceberg',
  'catalog-type'    = 'hadoop',
  'warehouse'       = 's3a://my-bucket/warehouse',
  'property-version'= '1'
);
USE CATALOG iceberg_catalog;
CREATE DATABASE analytics;
USE analytics;

CREATE TABLE events (
  id     BIGINT,
  payload STRING,
  ts     TIMESTAMP(6)
) WITH (
  'format-version' = '2',
  'write.upsert.enabled' = 'true'
);
-- Iceberg using Hive Metastore as catalog backend
CREATE CATALOG iceberg_hive WITH (
  'type'         = 'iceberg',
  'catalog-type' = 'hive',
  'uri'          = 'thrift://hive-metastore:9083',
  'warehouse'    = 's3a://my-bucket/warehouse'
);
-- Iceberg using JDBC as catalog backend (PostgreSQL)
CREATE CATALOG iceberg_jdbc WITH (
  'type'         = 'iceberg',
  'catalog-type' = 'jdbc',
  'uri'          = 'jdbc:postgresql://postgres:5432/iceberg_catalog',
  'jdbc.user'    = 'iceberg',
  'jdbc.password'= 'secret',
  'warehouse'    = 's3a://my-bucket/warehouse'
);

🗃 Apache Paimon

Paimon is a streaming data lakehouse format with native Flink integration. It supports primary-key tables with upserts, partial updates, and changelog generation.

Prerequisites

  • paimon-flink-1.19-0.8.jar in /opt/flink/lib/
CREATE CATALOG paimon_catalog WITH (
  'type'      = 'paimon',
  'warehouse' = 's3://my-bucket/paimon'
);
USE CATALOG paimon_catalog;
CREATE DATABASE IF NOT EXISTS ods;
USE ods;

-- Primary key table with changelog
CREATE TABLE orders (
  order_id BIGINT,
  status   STRING,
  amount   DECIMAL(18,2),
  updated_at TIMESTAMP(3),
  PRIMARY KEY (order_id) NOT ENFORCED
) WITH (
  'bucket'            = '4',
  'changelog-producer'= 'lookup'
);

🗄 JDBC Catalog

The JDBC Catalog reads table metadata directly from a relational database's information schema. Table and column definitions stay in sync with the database automatically.

CREATE CATALOG pg_catalog WITH (
  'type'                 = 'jdbc',
  'default-database'     = 'mydb',
  'username'             = 'flink_user',
  'password'             = 'secret',
  'base-url'             = 'jdbc:postgresql://postgres:5432'
);
USE CATALOG pg_catalog;
USE mydb;
SHOW TABLES;   -- shows PostgreSQL tables in mydb
ℹ
JDBC Catalog vs JDBC connector The JDBC Catalog manages table metadata (CREATE TABLE definitions pulled from the DB). The JDBC connector manages data reads and writes. You need both the catalog JAR and the connector JAR for full read/write access.

☁ AWS Glue Data Catalog

On Amazon Managed Apache Flink (formerly Kinesis Data Analytics), the AWS Glue catalog is available as a Hive-compatible catalog.

-- On Amazon Managed Apache Flink only
CREATE CATALOG glue_catalog WITH (
  'type'          = 'hive',
  'hive-conf-dir' = '/glue/confs',
  'default-database' = 'default'
);
⚠
Managed Flink specifics On Amazon Managed Apache Flink, the Glue connector is bundled. Refer to the AWS documentation for the exact configuration for your runtime version.

⟨/⟩ SQL Reference

StatementDescription
SHOW CATALOGSList all registered catalogs in the session
SHOW CURRENT CATALOGDisplay the currently active catalog
USE CATALOG nameSwitch the current catalog
SHOW DATABASESList databases in current catalog
CREATE DATABASE IF NOT EXISTS dbCreate a new database
USE dbSwitch the current database
SHOW TABLESList tables in current catalog.database
SHOW VIEWSList views in current catalog.database
DESCRIBE table_nameShow column names, types, and attributes
SHOW CREATE TABLE nameShow the DDL that created the table
DROP TABLE IF EXISTS nameRemove a table from the catalog
DROP CATALOG IF EXISTS nameRemove a registered catalog (does not delete data)
ALTER TABLE name RENAME TO newRename a table (persistent catalogs only)

⊞ Status Bar

The bottom status bar always shows the current catalog and database context: catalog: default_catalog.default. This is updated automatically whenever you run USE CATALOG or USE database_name. It provides a constant reminder of where unqualified table references will resolve.

💡
Always confirm your context Before running DDL like CREATE TABLE, check the status bar to ensure you're in the correct catalog and database. A table created in the wrong context may not be visible to your pipeline queries.

① Tutorial — Register a Hive Metastore Catalog

1

Deploy JARs

Copy flink-sql-connector-hive-3.1.3_2.12-1.19.jar and hive-exec-3.1.3.jar to /opt/flink/lib/ on all containers.

2

Mount hive-site.xml

Copy your hive-site.xml to /opt/flink/conf/hive-site.xml on the SQL Gateway container. The key property is hive.metastore.uris=thrift://hive-metastore:9083.

3

Open Catalog Manager in Studio

Click ⊕ Catalogs → Register Catalog → Type: Hive Metastore. Enter name: hive_catalog, Hive conf dir: /opt/flink/conf, Hive version: 3.1.3.

4

Click Register

Studio generates and runs the CREATE CATALOG SQL. The sidebar refreshes and shows the Hive catalog with all its databases and tables.

5

Switch to the catalog

Click hive_catalog in the list (or USE CATALOG hive_catalog) and then use the sidebar to browse Hive tables. Click any table name to insert it into the editor.

② Tutorial — Create an Iceberg Table on MinIO

1

Configure S3A for MinIO

Add to your Flink config.yaml: fs.s3a.endpoint: http://minio:9000, fs.s3a.access.key: minioadmin, fs.s3a.secret.key: minioadmin, fs.s3a.path.style.access: true.

2

Add Iceberg JAR

Copy iceberg-flink-runtime-1.19-1.6.1.jar to /opt/flink/lib/ on all containers and restart.

3

Register catalog in Studio

Open ⊕ Catalogs → Register Catalog → Type: Iceberg → Catalog type: hadoop, Warehouse: s3a://my-bucket/warehouse. Click Register.

4

Create a table

In the SQL editor:
USE CATALOG iceberg_catalog;
CREATE DATABASE analytics;
USE analytics;
CREATE TABLE events (id BIGINT, payload STRING, ts TIMESTAMP(3)) WITH ('format-version'='2');

5

Write to the table

Use a Pipeline Manager datagen → Iceberg sink pipeline or write directly: INSERT INTO events SELECT * FROM source_table;. Confirm: SELECT * FROM events LIMIT 10;
