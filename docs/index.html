ℹAbout Str:::lab Studio

Str:::lab Studio is a browser-based SQL Studio that connects to any Flink SQL Gateway. It provides a complete development, monitoring, and administration experience — without the overhead of a full platform deployment. Works with local Docker, Kubernetes, and any self-hosted Flink cluster.

What it is

A zero-dependency single-file HTML application served by nginx. All SQL execution is driven by the Flink SQL Gateway REST API. Results stream live into the browser. Session state is managed client-side in localStorage.

Who it's for

Streaming data engineers and platform teams who want a proper IDE experience for Apache Flink SQL — query editing, live result streaming, real-time job monitoring, checkpoint tracking, and professional reporting — all from a browser.

Open source

Created by Nestor A. A — open source under MIT. Community contributions welcome at github.com/coded-streams/strlabstudio.

Kube Operator

The Str:::lab Studio Kube Operator manages Studio deployments as first-class Kubernetes CRDs. Install via Helm from strlabstudio-operator.

Views vs UDFs — what's the difference?

ObjectHow to createWhere it appearsHow to use
ViewCREATE TEMPORARY VIEW name AS SELECT …SHOW VIEWS · Library "◫ Session Views" (teal)SELECT * FROM name — queryable like a table
JAR UDFCREATE FUNCTION name AS 'com.…' LANGUAGE JAVASHOW USER FUNCTIONS · Library "⨍ User-Defined" (blue)SELECT name(col) FROM … — callable in expressions
Built-inPre-registered by FlinkSHOW FUNCTIONS · Library "⨍ Built-in" (grey)SELECT json_value(col, '$.x') FROM …
ℹ
Correct Flink-native alternative to LANGUAGE SQL Use CREATE TEMPORARY VIEW to encapsulate any SQL expression and make it reusable by name across queries. Views are session-scoped, appear in SHOW VIEWS, and can be queried exactly like tables. This is the approach Flink itself recommends — there is no LANGUAGE SQL in Flink.

UDF Types

f(x)

Scalar (UDF)

One row in → one value out. Classification, masking, format conversion. Written in Java, Scala, or Python.

⇶

Table (UDTF)

One row in → many rows out. Used with CROSS JOIN LATERAL TABLE(...) to explode arrays or parse delimited fields.

∑

Aggregate (UDAGG)

Many rows in → one value out. Maintains an accumulator per group. Supports retraction for streaming tables.

⇄

Async Table

Non-blocking external enrichment. Issues many concurrent requests without blocking the pipeline.

Tab Reference

📚 Library tab

Click ⟳ Refresh to run SHOW FUNCTIONS, SHOW USER FUNCTIONS, and SHOW VIEWS in parallel. Results appear in three sections:

  • ◫ Session Views (teal) — views from SHOW VIEWS. Click to insert SELECT * FROM viewname.
  • ⨍ User-Defined Functions (blue) — JAR functions from SHOW USER FUNCTIONS. Click to insert name() at cursor.
  • ⨍ Built-in Functions (grey) — all Flink built-ins. Search and filter across all three sections simultaneously.

⬆ Upload JAR tab

Uploads your JAR to the Flink JobManager via POST /jars/upload. The JobManager REST URL section shows the Full URL Override and Port (auto) inputs on a single aligned row. Live progress bar tracks the upload. JARs on the cluster list with Use → and Delete buttons.

⚠
Port 8081 vs 8083 The JobManager REST API (port 8081, JAR uploads) is separate from the SQL Gateway (port 8083, SQL execution). Ensure /jobmanager-api/ is proxied in your nginx config.

⬡ Maven / Gradle tab

Generates a complete pom.xml or build.gradle for your UDF project. Fill in Group ID, Artifact ID, Flink version, Java version, and any extra dependencies. Preview updates live. Copy button included.

  • VALIDATION ERROR → the relevant validation line, not the full stack trace.
  • FunctionLanguage error → "Invalid LANGUAGE value. Flink only supports JAVA, SCALA, PYTHON."

🎨Colour Describe New — v1.0.23

Colour Describe lets you visually highlight result rows in real time based on user-defined field conditions — as rows stream in from a live Flink query. Rules are fully configurable: pick any column, any operator, any value, and any color. Every matching row is highlighted immediately and continues to be highlighted as new rows arrive.

🎨
Two highlighting modes exist side-by-side ✦ Auto Colour applies built-in semantic rules based on column name patterns (status, risk_score, latency_ms…) — one click, no configuration. 🎨 Colour Describe lets you define your own rules for any column in any query. Both buttons are in the Results toolbar.

Activating Colour Describe

Click 🎨 Colour Describe in the Results toolbar. This opens a configuration modal — it does not immediately apply any colours. You must configure at least one rule and click ⚡ Apply & Activate before any highlighting occurs. When active, the button reads 🎨 Colour Describe ●. Clicking it again (or clicking ✕ Turn off in the legend below the table) clears all highlighting instantly and restores default row styling.

Modal workflow

1

Select a query slot

A dropdown lists all active result slots — live streaming queries and completed queries with rows. Each entry shows the query label, live/done status (🔴 LIVE or ⬜ done), and row count. Select the slot you want to apply highlighting to. All column names of that slot auto-populate the Field dropdown.

2

Build a rule

Fill in the four parts of a rule:

PartOptions
FieldAny column name from the selected query slot
Operator== != > >= < <= contains starts ends regex
ValueString or numeric. Numeric operators auto-detect when both sides parse as numbers.
LabelShort description shown in the legend (e.g. "CRITICAL risk")
3

Pick a color

Eight preset swatches cover the most common semantic meanings: red (critical), amber (warning), yellow (caution), teal (ok), blue (info), purple (notice), green (success), orange (highlight). A custom color picker and live hex preview let you choose any color.

4

Choose a style mode

  • Row background — fills the entire row with a translucent tint of the chosen color.
  • Left border accent — adds a 4px solid left border in the chosen color, with a lighter background tint.
  • Text color — changes all text in the row to the chosen color.
5

Add, reorder, delete rules

Click ＋ Add Rule to add to the list. Rules are evaluated top-to-bottom — first match wins per row. Use the ↑↓ buttons to reorder. Click × to delete. There is no limit on the number of rules.

6

Apply & Activate

Click ⚡ Apply & Activate. All existing rows in the selected slot are immediately re-coloured. New rows are coloured as they stream in. A colour legend appears below the result table showing each rule's colour dot and label. The legend includes a ✕ Turn off button.

PDF report integration

When Colour Describe is active and a session report is generated (Performance → Report → Generate PDF), the report automatically includes a Colour Describe Rules section with: rule definitions (field, operator, value, color, style mode, label) and a sample of up to 20 matched rows from the active slot. If Colour Describe is toggled off before generating the report, this section is omitted entirely.

ℹ
Report status indicator The report status line shows 🎨 Colour Describe included when rules are active, so you can confirm before generating the PDF.

Integration point for results.js

To enable live row highlighting in the Results table, call this function in results.js after each <tr> is created:

// In the row rendering loop inside renderResults():
applyColorDescribeToRow(rowEl, row.fields, state.resultColumns);

Function signature:

applyColorDescribeToRow(
  rowEl:    HTMLTableRowElement,  // the <tr> element
  rowData:  any[],                // cell values in column order
  columns:  { name: string }[]    // column definitions
)

The function is a no-op when window.colorDescribeActive is false, so it is safe to call unconditionally on every row render.
